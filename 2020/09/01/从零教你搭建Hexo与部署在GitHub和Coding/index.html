发表于2020-09-01|更新于2020-09-01|Hexo
|阅读量:

前言：
经历了Wordpress动态博客系统的折腾后，开始转向静态博客，故学习了Hexo这个框架。期间遇到了不少阻碍，总结一下心得。

什么是Hexo？


轻易集成。快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。只需一条指令即可部署到 GitHub Pages, Heroku 或其他平台。强大的 API 带来无限的可能，与数种模板引擎（EJS，Pug，Nunjucks）和工具（Babel，PostCSS，Less/Sass）

准备环境

  • Node.js
  • Git

安装Node.js

  • Mac：使用 Homebrew 或 MacPorts 安装。
  • Windows：可以直接去官方下载并安装Node.js
  • Linux：（DEB/RPM-based）：从 NodeSource 安装。

安装Git

  • Mac：使用 Homebrew, MacPorts 或者下载 安装程序。
  • Windows：可以去官方直接下载并安装 git.
  • Linux (Ubuntu, Debian)：sudo apt-get install git-core
  • Linux (Fedora, Red Hat, CentOS)：sudo yum install git-core

安装成功之后了，Windows打开git bash，Mac（Linux）打开终端。输入相应命令查看是否安装成功，若是安装成功会有相应的版本号。

使用Coding

由于众所周知的原因，使用GitHub对于国内访问不太友好，所以我们就要找其他的方案，Coding在卖给腾讯后从单纯git服务变成了项目管理，默认为团队形式，5人以下的团队免费使用。
Coding Pages服务免费了，但是需要实名认证。

关于Coding，还能利用免费的工作台（5人以下）进行web端部署Hexo，实现何时何地写博客并上传，后续继续更新，请继续关注。
